# 쿠키 vs 세션 스토리지 vs 로컬 스토리지

JWT토큰을 저장하는 곳은 크게 웹 스토리지와 쿠키로 나눌 수 있고, 웹 스토리지는 세션, 로컬로 나눌 수 있다.



### 쿠키 vs 웹 스토리지

웹 스토리지 방식은 쿠키와는 달리 모든 HTTP 요청에서 데이터를 주고받을 필요가 없다.

| 쿠키                                          | 웹 스토리지             |
| --------------------------------------------- | ----------------------- |
| 4KB만 저장 가능                               | 약 5MB 까지 저장 가능   |
| 매번 HTTP 요청 헤더에 붙여 전송 (트래픽 낭비) | 서버로 전송할 필요 없음 |
| 보안에 취약                                   | 상대적으로 보안에 안전  |
| 20개 정도로 개수 제한                         | 개수 제한 없음          |



### 세션 스토리지 vs 로컬 스토리지

- 세션 스토리지

  브라우저가 종료되면 자동으로 제거된다.

- 로컬 스토리지

  브라우저에 반영구적으로 데이터를 저장한다. 브라우저를 종료해도 데이터가 유지된다.
